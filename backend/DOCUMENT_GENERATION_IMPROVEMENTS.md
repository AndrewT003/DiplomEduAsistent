# Покращення генерації документів

## Зміни від 23.03.2026

### ✅ Що було покращено:

#### 1. **Автоматичне витягування теми заняття**
- Додана функція `extract_topic_from_context()` в `generators.py`
- Тема автоматично витягується з контексту документа
- Використовується як заголовок у згенерованих документах
- Зберігається в БД в полі `topic` таблиці `materials`

#### 2. **Правильне оформлення документів**
- **Шрифт**: Times New Roman для всіх елементів
- **Розмір**: 14pt для основного тексту, 16pt для заголовків
- **Колір**: Чорний (#000000) для всього тексту
- **Вирівнювання**:
  - Заголовки - по центру
  - Текст - по ширині (justify)
- Оформлення застосовується до:
  - Конспектів
  - Тестів
  - Флеш-карток
  - Глосаріїв
  - Звітів валідації

#### 3. **Технічні зміни**

**generators.py**:
- `generate_material()` тепер повертає `dict` з ключами `content` і `topic`
- Додана функція `extract_topic_from_context()` для витягування теми

**docx_exporter.py**:
- Додана функція `set_times_new_roman_14pt()` для форматування тексту
- Додана функція `apply_font_to_document()` для застосування шрифту до всього документа
- `export_to_docx()` приймає параметр `topic` для динамічного заголовка
- Оновлена `export_validation_to_docx()` для використання Times New Roman

**main.py**:
- Оновлений endpoint `/generate/{doc_id}` для збереження теми
- Оновлений endpoint `/download/{doc_id}/{material_type}` для використання збереженої теми

**migration_add_topic_to_materials.sql**:
- Нова міграція для додавання поля `topic` до таблиці `materials`

### 📝 Інструкція по застосуванню міграції:

```bash

psql -h your-supabase-host -U postgres -d postgres -f migration_add_topic_to_materials.sql
```

Або скопіюйте вміст файлу в Supabase SQL Editor.

### 🎯 Результат:

Тепер всі згеновані документи:
- Мають тему витягнуту з контексту (замість "Документ {id}")
- Оформлені в Times New Roman 14pt
- Весь текст чорний
- Заголовки вирівняні по центру
- Основний текст вирівняний по ширині
- Виглядають професійно та відповідають вимогам
